Режим вывода текста в результате валидации.

##  __Члены
General| 0|  Режим по умолчанию, показывает тип сообщения (ошибка,
предупреждение или информация), имя и тип объекта (например, имя расширения),
а также текст сообщения. В строке формата можно указать
validationResult.ToString("G").  
---|---|---  
Detailed| 1|  Детализированный вывод, где для каждого сообщения помимо текста
из General отображаются также детали при их наличии. В качестве деталей может
выступать стек-трейс исключений. Рекомендуется использовать именно этот режим
вывода всегда, когда результат валидации может содержать исключения, по
которым необходимо вывести стек-трейс. В строке формата можно указать
validationResult.ToString("D").  
Message| 2|  Выводится только текст сообщения. Информация в таком виде может
выводиться для пользователя. В строке формата можно указать
validationResult.ToString("M").  
TypeAndMessage| 3|  Указывается тип сообщения (ошибка, предупреждение или
информация) и текст сообщения. В строке формата можно указать
validationResult.ToString("TM").  
DetailedWithoutType| 4|  Детализированный вывод, где для каждого сообщения
помимо текста из General отображаются также детали при их наличии. При этом не
выводится тип сообщения (ошибка, предупреждение или информация), в отличие от
Detailed. В качестве деталей может выступать стек-трейс исключений.
Рекомендуется использовать именно этот режим вывода всегда, когда результат
валидации может содержать исключения, по которым необходимо вывести стек-
трейс, причём тип сообщения уже известен, например, по уровню логирования в
логе (сообщения об ошибка выводятся с уровнем логирования Error, и др.). В
строке формата можно указать validationResult.ToString("DT").
